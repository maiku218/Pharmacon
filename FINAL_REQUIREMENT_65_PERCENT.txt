3. APIs (Application Programming Interfaces)
================================================================================

3.1 RESTful Endpoints

    PUBLIC API:
    - GET /api/products
        Description: Get all available products (stock > 0)
        Response: JSON array of products
        
    - GET /search_product?q=<query>
        Description: Search products by name or barcode
        Response: JSON array of matching products
        
    - GET /get_product/<barcode>
        Description: Get product details by barcode
        Response: JSON product object
        
    - GET /api/search_by_name?q=<query>
        Description: Autocomplete search by product name
        Response: JSON array of matching products

    CASHIER API:
    - POST /complete_sale
        Description: Process completed sale transaction
        Input: JSON {items: [], total: float}
        Response: JSON {success, receipt_number, total, items, date}
        
    - POST /cashier_logout
        Description: Logout cashier and record logout time
        Response: JSON {success}

    ADMIN API:
    - GET /api/notifications
        Description: Get low stock and expiring product alerts
        Response: JSON {low_stock: [], expiring: []}
        
    - GET /active_cashiers
        Description: Get all cashiers with online status
        Response: JSON array of cashiers

3.2 JSON Data Format
    - All APIs return JSON responses
    - Uses Flask jsonify() for proper JSON encoding
    - Includes error handling for failed requests

6. DATABASE GATEWAY
================================================================================

6.1 Database Management System
    - MySQL (via XAMPP)
    - MySQL will be hosted locally using XAMPP package
    - Database name: pharmacon

6.2 Database Driver
    - Flask-MySQLdb (Python MySQL connector)
    - Provides Python API for MySQL operations

6.3 Database Schema
    The system uses the following tables:
    
    a) admins
       - id (INT, PRIMARY KEY, AUTO_INCREMENT)
       - username (VARCHAR)
       - password (VARCHAR, hashed)
       - full_name (VARCHAR)
       - created_at (DATETIME)
    
    b) cashiers
       - id (INT, PRIMARY KEY, AUTO_INCREMENT)
       - full_name (VARCHAR)
       - username (VARCHAR)
       - password (VARCHAR, hashed)
       - status (VARCHAR)
       - created_at (DATETIME)
    
    c) cashier_activity
       - id (INT, PRIMARY KEY, AUTO_INCREMENT)
       - cashier_id (INT, FOREIGN KEY)
       - login_time (DATETIME)
       - logout_time (DATETIME)
       - ip_address (VARCHAR)
    
    d) admin_activity
       - id (INT, PRIMARY KEY, AUTO_INCREMENT)
       - admin_id (INT, FOREIGN KEY)
       - action (VARCHAR)
       - ip_address (VARCHAR)
       - details (TEXT)
       - activity_time (DATETIME)
    
    e) categories
       - id (INT, PRIMARY KEY, AUTO_INCREMENT)
       - category_name (VARCHAR)
       - description (TEXT)
    
    f) products
       - id (INT, PRIMARY KEY, AUTO_INCREMENT)
       - product_name (VARCHAR)
       - barcode (VARCHAR)
       - category_id (INT, FOREIGN KEY)
       - type (VARCHAR) - Medical/Non-Medical
       - price (DECIMAL)
       - stock (INT)
       - expiration_date (DATE)
       - created_at (DATETIME)
    
    g) stock_movements
       - id (INT, PRIMARY KEY, AUTO_INCREMENT)
       - product_id (INT, FOREIGN KEY)
       - movement_type (VARCHAR) - IN/OUT
       - quantity (INT)
       - reason (VARCHAR)
       - movement_date (DATETIME)
    
    h) sales
       - id (INT, PRIMARY KEY, AUTO_INCREMENT)
       - receipt_number (VARCHAR)
       - cashier_id (INT, FOREIGN KEY)
       - total_amount (DECIMAL)
       - sale_status (VARCHAR)
       - product_type (VARCHAR) - Medical/Non-Medical
       - sale_date (DATETIME)
    
    i) sale_items
       - id (INT, PRIMARY KEY, AUTO_INCREMENT)
       - sale_id (INT, FOREIGN KEY)
       - product_id (INT, FOREIGN KEY)
       - quantity (INT)
       - price (DECIMAL)

6.4 Database Configuration
    - Host: localhost
    - User: root
    - Password: (empty - default XAMPP)
    - Database: pharmacon
